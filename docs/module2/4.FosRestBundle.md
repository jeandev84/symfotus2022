### FosRestBundle


1. FosRestBundle : Зачем он нужен ?
```
Независимость кода контроллеров от формата сериализации
Возможность сериализовызать Enity-классы вместе DTO в простых случаях
Обработка тела запроса
Dependency Injection DTO в метод контроллера
Есть централизованная обработка исключений ( лучше использовать штатный механизм )
```


2. Packages
```
composer require sensio/framework-extra-bundle ( работа с представлениями )
composer require friendsofsymfony/rest-bundle  ( продвинутная обработка тела запроса )
composer require jms/serializer-bundle         ( сериализатор / десериализатор )
```


3. REST (Архитектурный стиль в каким образом, правильно построить приложение)
Representation State Transfer
```
Какие у него требования ?
1. Модель клиент-сервер
2. Отсуствие состояния (stateless)
3. Кэширование ( определяется сервером )
4. Единообразие интерфейса
   - Идентификация ресурсов
   - Манипуляция через представление
   - Сообщение описывает, как его обрабатывать
   - HATEOAS ( Hypermedia as the Engine of Application State )
5. Слои
Клиент не должен зависит от того какой слой прислали ему ответ
Клиент может отправить запрос на какой-то микросервис 
и этот микросервис должен объязательно ответить клиенту

6. Код по требованию
```


4. REST Over HTTP 
```
Некорректный URL                               | 404
Запрос несуществующей страницы при пагинации   | 204
Вызов Deprecated-метода                        | 410
Race condition                                 | 409
Синтаксическая ошибка в теле запроса           | 400
Логическая ошибка в теле запроса               | 422
Постановка асинхронной задачи в очередь        | 202
Сохранение новой сущности                      | 201
Удаление уже удалённой сущности                | 404/200
Доступ к сущности запрещён                     | 404
```
